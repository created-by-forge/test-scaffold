## Tech Stack

- **Framework:** React + TypeScript
- **Build:** Vite
- **Styling:** Tailwind CSS v4 (CSS-based config in `src/index.css`, no tailwind.config file)
- **Routing:** React Router DOM v7
- **State:** Zustand (stores in `src/stores/`)
- **DB:** Supabase (client in `src/lib/supabase.ts`)
- **Hosting:** Vercel
- **Utilities:** clsx + tailwind-merge via `cn()` in `src/lib/utils.ts`

## Coding Conventions

- **File naming:** kebab-case: `calls-page.tsx`, `data-service.ts`
- **Imports:** Always use `@/` path alias (maps to `src/`)
- **Components:** Named exports, inline props interfaces, `cn()` for conditional classes
- **State:** Zustand stores named `{domain}-store.ts`
- **Styling:** Tailwind v4 design tokens from `@theme` block, dark-first
- **Database:** snake_case tables, UUID PKs, TIMESTAMPTZ, RLS on all tables
- **Migrations:** Sequential numbered (001_, 002_)

## Rules

### Never
- Never import with relative paths — always use `@/`
- Never use raw color values — use design system tokens
- Never add npm dependencies without asking first
- Never modify applied migrations — create a new one
- Never store secrets in code — use VITE_ prefix for client, plain for serverless
- Never commit `.env` files

### Always
- Always run `npm run build` to validate before considering work done
- Always use `cn()` for conditional Tailwind classes
- Always use TIMESTAMPTZ for new database columns
- Always add RLS policies when creating new Supabase tables
- Always use named exports (not default exports)
